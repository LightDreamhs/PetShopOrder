# 前后端联调指南：一场完整的购物旅程

本指南按时间顺序模拟一场完整的购物流程，标注每一步在哪个页面操作、调用什么 API、另一端如何看到效果。适用于地图选点、地址管理、配送计算、新订单通知等功能接入后的当前版本。

## 环境与服务

项目包含两个前端 + 一个后端：

- **H5**（顾客端）：`http://localhost:3000` — Vue 3 + Vant 4
- **Admin**（商家端）：`http://localhost:3001` — Vue 3 + Element Plus
- **Backend**：`http://localhost:8080/api/` — Spring Boot + Sa-Token
- **上传文件访问**：`http://localhost:8080/uploads/`
- **地图服务**：腾讯地图 JavaScript API GL（H5 与 Admin 共用一个 Key，配在各自 `.env.development` 的 `VITE_TMAP_KEY`）

> **提示**：后端 CORS 已允许 `192.168.x.x:3000/3001`，可用手机浏览器访问 `http://<电脑局域网 IP>:3000` 真实体验 H5 地图选点与定位。

## 第一幕：商家开店准备

### 步骤 1：商家登录 Admin

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/login` |
| **操作** | 输入用户名密码，点 **登录** |
| **API** | `POST /api/admin/auth/login` |
| **验证** | 跳转到订单管理页（默认首页），侧边栏显示角色可见的菜单 |

### 步骤 2：配置系统参数（BOSS/MANAGER 角色）

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/settings`（系统配置） |
| **操作** | 配置配送规则、接单时段、Webhook 通知、收款码 |
| **API** | `GET /api/admin/system-config` → `PUT /api/admin/system-config` |
| **验证** | 保存成功后刷新页面，配置值回显正确 |

本步重点配置项：

- **店铺位置**（必填）：点 **地图选点**，在弹出的地图上拖动定位或搜索地点，确认后保存店铺经纬度。这是配送距离计算的起点，**未配置会导致下单时距离计算报错**。
- **配送半径**、**起送价**、**运费规则**（免运费 / 固定 / 分段）
- **接单时段**：限制可下单的时间范围
- **群机器人 Webhook**：支持飞书或企业微信，下单后自动推送通知
- **收款二维码**：下单成功页展示给顾客

### 步骤 3：新增商品 + SKU

> **分类模型已简化**：商品只有固定的两种类型——「用品」（GOODS）和「服务」（SERVICE），无子分类。

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/products` — 点 **新增商品** |
| **操作** | 填写商品名、上传封面图、选类型（用品 / 服务）、是否支持配送、排序；添加 SKU（规格名、原价、会员价、排序） |
| **API** | `POST /api/admin/files/upload`（上传图片）→ `POST /api/admin/products`（创建商品） |
| **验证** | 商品列表出现新商品，状态为「上架」 |
| **H5 联动** | H5 HomePage 刷新后能看到新商品，出现在对应的「用品」或「服务」Tab 下 |

### 步骤 4：设置会员等级

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/members`（会员管理）— 会员等级区域 |
| **操作** | 系统预置 4 档会员等级，折扣仅对服务类商品生效，可增删改 |
| **API** | `POST /api/admin/member-levels` |
| **验证** | 等级列表出现新等级 |

### 步骤 5：录入会员 + 绑定手机号

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/members` — 会员列表区域 |
| **操作** | 新增会员，选等级，填写手机号（可多个），备注 |
| **API** | `POST /api/admin/members` |
| **验证** | 会员列表出现新会员 |
| **H5 联动** | 该手机号用户在 H5 登录时，`GET /api/app/member/profile` 返回会员信息，购物车和结算页显示会员折扣价 |

## 第二幕：顾客逛店下单

### 步骤 6：顾客登录 H5

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/login` |
| **操作** | 输入手机号 → 获取验证码 → 输入验证码 → 登录（验证码固定 `1234`） |
| **API** | `POST /api/app/auth/sms-code` → `POST /api/app/auth/login` |
| **验证** | 跳转到首页，顶部显示脱敏手机号 |
| **会员联动** | 手机号命中会员时，首页出现会员徽章，商品显示会员价 |
| **测试手机号** | 初始数据有 6 个会员测试号：`13800001111`/`222`（2000 档）、`13900003333`（1000 档）、`13700004444`/`555`（500 档）、`13600006666`（1000 档）、`13500007777`（500 档）、`13400008888`/`999`（5000 档） |

### 步骤 7：浏览商品 + 加购物车

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/`（首页） |
| **API** | `GET /api/app/products`（加载全部商品） |
| **操作** | Tab 切换「用品」/「服务」，选规格后加入购物车 |
| **注意** | 购物车是纯前端 Pinia + localStorage，**不调 API** |
| **验证** | CartBar 数量徽标与金额更新正确 |

### 步骤 8：进入结算页

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/cart` → 点 **去结算** → H5 `/checkout` |
| **API** | `POST /api/app/cart/calculate`（无地址，仅校验起送金额） |
| **验证** | 结算页展示商品明细、原价 / 会员折扣、配送开关、费用明细 |

> **配送开关的启用条件**：后端 `calculate` 接口始终返回 `deliveryCheck`，开关根据 `reachedMinAmount` 判断是否可点。当用品原价未达起送价时，开关禁用并提示「用品原价还差 ¥X 起送」。

### 步骤 9：选择收货地址

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/checkout` — 开启 **需要配送** 开关 |
| **API** | `GET /api/app/addresses`（拉取已保存地址） |
| **验证** | 地址区域显示已保存地址列表，默认地址置顶 |

配送地址选择逻辑：

- **已有地址**：直接点选列表中的地址，**不调用地图 API**
- **无地址**：自动弹出地图选点页，引导用户添加第一个地址
- **新增地址**：点列表底部 **新增地址**，打开地图选点

### 步骤 10：地图选点新增地址

| 项目 | 内容 |
|---|---|
| **页面** | H5 地图选点全屏弹窗（AddressPicker 组件） |
| **API** | 腾讯地图 `Geocoder` / `Search.searchRegion`（前端直连腾讯，不经后端） |
| **操作** | 拖动地图选点、搜索地点、点定位按钮获取当前位置；底部勾选「保存为常用地址」并选标签（家 / 公司 / 学校 / 其他） |
| **保存 API** | 点 **选择此地点** 时，若勾选保存则调用 `POST /api/app/addresses` |
| **验证** | 返回结算页，新地址出现在列表中并自动选中 |

> **配额说明**：腾讯地图地点搜索 API 免费配额为每日 200 次。常用地址机制确保老用户下单选地址不消耗配额，仅新增地址时调用。

### 步骤 11：提交订单

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/checkout` — 点 **提交订单** |
| **API** | `POST /api/app/orders` |
| **验证** | 跳转到 `/order/success`，显示订单号、总金额、收款二维码。若配置了 Webhook，后端异步推送通知 |
| **配送计算** | 后端用店铺坐标与收货地址坐标做 Haversine 距离计算，结合配送半径、分段运费规则算出运费 |

### 步骤 12：查看我的订单

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/orders`（我的订单） |
| **API** | `GET /api/app/orders`（分页）→ 点某单进详情 `GET /api/app/orders/{id}` |
| **验证** | 订单列表与详情展示正确：商品快照、金额、配送地址、配送距离、处理状态 |

### 步骤 13：管理收货地址（可选）

| 项目 | 内容 |
|---|---|
| **页面** | H5 `/address/manage`（结算页点 **管理** 进入） |
| **API** | `GET /api/app/addresses`、`PUT /api/app/addresses/{id}/default`、`DELETE /api/app/addresses/{id}` |
| **操作** | 设默认地址、删除地址、新增地址 |
| **验证** | 操作后列表实时更新，结算页同步刷新 |

## 第三幕：商家处理订单

### 步骤 14：接收新订单通知

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/orders`（订单管理） |
| **API** | 每 15 秒轮询 `GET /api/admin/orders/new-count?since=<上次检查时间>` |
| **效果** | 有新订单时，右下角弹出 ElNotification「有新订单来了」，点击跳到第一页；订单列表自动刷新 |

> **说明**：采用轻量轮询方案替代 WebSocket，无需额外中间件，刷新页面即生效。

### 步骤 15：处理订单

| 项目 | 内容 |
|---|---|
| **页面** | Admin `/orders` — 点订单查看详情 |
| **API** | `GET /api/admin/orders/{id}` → `PUT /api/admin/orders/{id}/processed` |
| **操作** | 确认处理完毕后标记为「已处理」 |
| **验证** | 订单状态变为「已处理」 |
| **H5 联动** | 顾客在 H5 订单详情刷新后看到状态更新 |

## 第四幕：其他联调场景

### 场景 A：商品上下架与改价

| Admin 操作 | API | H5 效果 |
|---|---|---|
| 商品 **下架** | `PUT /api/admin/products/{id}/status` body `{status:"OFF_SALE"}` | H5 首页不再显示，购物车中提示已下架 |
| 商品 **上架** | 同上 body `{status:"ON_SALE"}` | H5 首页重新出现 |
| 编辑 SKU 价格 | `PUT /api/admin/products/{id}` | H5 价格更新。已下过的订单不受影响（快照机制） |
| 删除商品 | `DELETE /api/admin/products/{id}` | H5 不再显示，已有订单快照不受影响 |

### 场景 B：配送规则变更

| Admin 操作 | API | H5 效果 |
|---|---|---|
| 修改店铺位置 / 配送半径 / 起送价 / 运费 | `PUT /api/admin/system-config` | H5 结算页重新计算，可能超出配送范围或起送价提示 |

### 场景 C：会员折扣变更

| Admin 操作 | API | H5 效果 |
|---|---|---|
| 修改会员等级折扣率 | `PUT /api/admin/member-levels/{id}` | H5 下次加载商品时，服务类会员价按新折扣计算 |

## 快速联调清单

按优先级排列，建议按此顺序逐项验证：

1. **商家登录** → Admin 能正常登录和退出
2. **配置店铺位置** → 地图选点保存成功，坐标回显正确
3. **新增商品** → Admin 创建后，H5 刷新能看到（对应类型 Tab 下）
4. **顾客登录** → H5 验证码登录成功
5. **浏览商品 + 加购 + 结算** → 金额计算正确（含会员折扣）
6. **配送开关启用** → 用品原价达起送价时开关可点，否则提示差额
7. **新增收货地址** → 地图选点 + 搜索 + 定位正常，保存为常用地址
8. **提交订单** → 配送距离与运费计算正确，下单成功，展示收款码
9. **新订单通知** → Admin 右下角 15 秒内弹出新订单提醒
10. **处理订单** → Admin 标记已处理，H5 刷新状态更新
11. **地址管理** → 设默认、删除、新增均正常
12. **角色权限** → STAFF 只能看订单，MANAGER 看不到系统配置和账号管理
